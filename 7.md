# ✅ **Concept (RARP Protocol Simulation)**

### 🔹 What is RARP?

**Reverse Address Resolution Protocol** is the reverse of ARP:

```text
MAC Address  →  IP Address
```

### 🔹 Why RARP is used?

- A device knows its **MAC address**
    
- It wants to find its **IP address**  
    👉 It sends a request to a RARP server
    

---

### 🔹 Real Working (Idea)

1. Client sends **MAC address** (broadcast)
    
2. RARP server checks table
    
3. Sends corresponding **IP address**
    

---

### 🔹 What we simulate

- Use **UDP (connectionless)**
    
- Client sends MAC address
    
- Server checks a predefined table
    
- Server replies with IP address

# 💻 **Java Program**

## 🔹 RARP Server (UDP)

```java
import java.net.*;
import java.util.HashMap;

public class RARPServer {
    public static void main(String[] args) {
        try {
            DatagramSocket socket = new DatagramSocket(5000);

            // Simulated RARP Table (MAC → IP)
            HashMap<String, String> table = new HashMap<>();
            table.put("AA:BB:CC:DD:EE:01", "192.168.1.1");
            table.put("AA:BB:CC:DD:EE:02", "192.168.1.2");
            table.put("AA:BB:CC:DD:EE:03", "192.168.1.3");

            byte[] buffer = new byte[1024];

            System.out.println("RARP Server is running...");

            // Receive request
            DatagramPacket request = new DatagramPacket(buffer, buffer.length);
            socket.receive(request);

            String mac = new String(request.getData(), 0, request.getLength());
            System.out.println("Received MAC: " + mac);

            String ip = table.getOrDefault(mac, "IP not found");

            // Send response
            byte[] sendData = ip.getBytes();
            DatagramPacket response = new DatagramPacket(
                    sendData,
                    sendData.length,
                    request.getAddress(),
                    request.getPort()
            );

            socket.send(response);
            socket.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
```

## 🔹 RARP Client (UDP)

```java
import java.net.*;
import java.util.Scanner;

public class RARPClient {
    public static void main(String[] args) {
        try {
            DatagramSocket socket = new DatagramSocket();
            Scanner sc = new Scanner(System.in);

            System.out.print("Enter MAC Address: ");
            String mac = sc.nextLine();

            byte[] sendData = mac.getBytes();
            InetAddress ip = InetAddress.getByName("localhost");

            // Send request
            DatagramPacket request = new DatagramPacket(sendData, sendData.length, ip, 5000);
            socket.send(request);

            // Receive response
            byte[] buffer = new byte[1024];
            DatagramPacket response = new DatagramPacket(buffer, buffer.length);
            socket.receive(response);

            String result = new String(response.getData(), 0, response.getLength());
            System.out.println("IP Address: " + result);

            socket.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
```

# 🧠 **Explanation (Simple)**

- Client sends MAC address via UDP
    
- Server looks into **RARP table (HashMap)**
    
- Sends corresponding IP
    
- If not found → returns "IP not found"
