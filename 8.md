# ✅ **Concept (Congestion Control)**

### 🔹 What is Congestion?

Congestion occurs when:

- Too many packets are sent into the network
    
- Network cannot handle the load  
    👉 Leads to **packet loss, delay, reduced throughput**
    

---

### 🔹 What is Congestion Control?

Mechanisms used to:

- Control traffic rate
    
- Avoid packet loss
    
- Improve performance
    

---

### 🔹 Example Algorithm

The most common is **TCP Congestion Control**

It includes:

- **Slow Start**
    
- **Congestion Avoidance**
    
- **Fast Retransmit**
    
- **Fast Recovery**

# 💻 **NS2 Simulation Program (TCL Script)**

```tcl
# Create simulator
set ns [new Simulator]

# Open trace file
set tracefile [open out.tr w]
$ns trace-all $tracefile

# Create nodes
set n0 [$ns node]
set n1 [$ns node]
set n2 [$ns node]

# Create links
$ns duplex-link $n0 $n1 1Mb 10ms DropTail
$ns duplex-link $n1 $n2 1Mb 10ms DropTail

# Set queue limit (to simulate congestion)
$ns queue-limit $n0 $n1 5

# Create TCP agent
set tcp [new Agent/TCP]
$ns attach-agent $n0 $tcp

# Create sink
set sink [new Agent/TCPSink]
$ns attach-agent $n2 $sink

# Connect TCP and sink
$ns connect $tcp $sink

# Create FTP application
set ftp [new Application/FTP]
$ftp attach-agent $tcp

# Start and stop simulation
$ns at 0.5 "$ftp start"
$ns at 4.5 "$ftp stop"

# End simulation
$ns at 5.0 "finish"

proc finish {} {
    global ns tracefile
    $ns flush-trace
    close $tracefile
    exec nam out.nam &
    exit 0
}

# Run simulation
$ns run
```

# 📊 **Performance Analysis**

From the trace file (`out.tr`), we evaluate:

### 🔹 1. Throughput

- Amount of data successfully delivered
    
- Increases after slow start
    

---

### 🔹 2. Packet Loss

- Occurs when queue is full
    
- Seen due to small queue limit
    

---

### 🔹 3. Delay

- Time taken for packet delivery
    
- Increases during congestion
    

---

### 🔹 4. Congestion Window Behavior

- Initially increases exponentially (**slow start**)
    
- Then linearly (**congestion avoidance**)
    
- Drops on packet loss
    

---

# 📌 **Expected Observation**

- At start → high growth (slow start)
    
- When congestion occurs → packet drop
    
- TCP reduces sending rate
    
- System stabilizes
