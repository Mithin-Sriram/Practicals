# ✅ **5(a) File Transfer using TCP Socket (Java)**

## 🔹 **Concept**

- Uses **TCP (reliable, connection-oriented)**
    
- Client reads a file → sends bytes to server
    
- Server receives bytes → writes to a new file
    

👉 This is basically how protocols like **FTP** work internally.

---

## 💻 **Server Program (Receive File)**

```java
import java.io.*;
import java.net.*;

public class FileServer {
    public static void main(String[] args) {
        try {
            ServerSocket serverSocket = new ServerSocket(5000);
            System.out.println("Server waiting for connection...");

            Socket socket = serverSocket.accept();
            System.out.println("Client connected");

            InputStream in = socket.getInputStream();
            FileOutputStream fos = new FileOutputStream("received_file.txt");

            byte[] buffer = new byte[1024];
            int bytesRead;

            while ((bytesRead = in.read(buffer)) > 0) {
                fos.write(buffer, 0, bytesRead);
            }

            System.out.println("File received successfully");

            fos.close();
            socket.close();
            serverSocket.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
```

---

## 💻 **Client Program (Send File)**

```java
import java.io.*;
import java.net.*;

public class FileClient {
    public static void main(String[] args) {
        try {
            Socket socket = new Socket("localhost", 5000);

            File file = new File("send_file.txt");
            FileInputStream fis = new FileInputStream(file);

            OutputStream out = socket.getOutputStream();

            byte[] buffer = new byte[1024];
            int bytesRead;

            while ((bytesRead = fis.read(buffer)) > 0) {
                out.write(buffer, 0, bytesRead);
            }

            System.out.println("File sent successfully");

            fis.close();
            socket.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
```

## 🧠 **Explanation (Simple)**

- Client reads file → sends bytes
    
- Server reads bytes → writes file
    
- TCP ensures:
    
    - No data loss
        
    - Correct order

# ⚠️ **5(b) Wireshark + FTP Username & Password**

I need to be clear here:  
Capturing someone’s credentials without permission is **not okay**.

But for **learning/lab purposes**, here’s the **conceptual explanation**.

---

## 🔹 **Concept**

- **Wireshark** is a packet analyzer
    
- **FTP** sends data in **plain text**
    
- That includes:
    
    - Username → `USER`
        
    - Password → `PASS`
        

---

## 🔍 **What Happens in FTP Traffic**

When logging in:

```
USER myname
PASS mypassword
```

These are visible in captured packets because:  
👉 FTP is **not encrypted**

---

## 🧪 **Lab Demonstration (Safe Way)**

Only do this on:

- Your own system OR
    
- Lab environment OR
    
- Permission-based network
    

### Steps:

1. Open Wireshark
    
2. Start capture on your network
    
3. Apply filter:
    

```
ftp
```

4. Perform FTP login (test server)
    
5. Look for packets containing:
    

```
USER
PASS
```

---

## 📌 **Key Learning Point**

- FTP is insecure ❌
    
- Modern protocols use:
    
    - **SFTP**
        
    - **FTPS**
        

These encrypt credentials ✔️
