# ✅ **Concept (TCP Chat Application)**

This program creates a **two-way communication (chat)** between client and server using TCP.

### 🔹 Key Idea:

- Both **client and server can send & receive messages simultaneously**
    
- Uses **TCP sockets (connection-oriented)**
    
- Requires **multithreading**:
    
    - One thread → send messages
        
    - One thread → receive messages
        

---

### 🔹 Why Threads?

Without threads:

- Program can either **send OR receive**, not both at same time  
    With threads:
    
- Chat becomes **full duplex (both directions simultaneously)**

# 💻 **Java Program**

## 🔹 Server Program

```java
import java.io.*;
import java.net.*;

public class ChatServer {
    public static void main(String[] args) {
        try {
            ServerSocket serverSocket = new ServerSocket(5000);
            System.out.println("Server waiting...");

            Socket socket = serverSocket.accept();
            System.out.println("Client connected!");

            BufferedReader in = new BufferedReader(
                    new InputStreamReader(socket.getInputStream()));
            PrintWriter out = new PrintWriter(socket.getOutputStream(), true);

            BufferedReader userInput = new BufferedReader(
                    new InputStreamReader(System.in));

            // Thread to receive messages
            new Thread(() -> {
                try {
                    String msg;
                    while ((msg = in.readLine()) != null) {
                        System.out.println("Client: " + msg);
                    }
                } catch (Exception e) {}
            }).start();

            // Sending messages
            String msg;
            while ((msg = userInput.readLine()) != null) {
                out.println(msg);
            }

            socket.close();
            serverSocket.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
```

## 🔹 Client Program

```java
import java.io.*;
import java.net.*;

public class ChatClient {
    public static void main(String[] args) {
        try {
            Socket socket = new Socket("localhost", 5000);
            System.out.println("Connected to server!");

            BufferedReader in = new BufferedReader(
                    new InputStreamReader(socket.getInputStream()));
            PrintWriter out = new PrintWriter(socket.getOutputStream(), true);

            BufferedReader userInput = new BufferedReader(
                    new InputStreamReader(System.in));

            // Thread to receive messages
            new Thread(() -> {
                try {
                    String msg;
                    while ((msg = in.readLine()) != null) {
                        System.out.println("Server: " + msg);
                    }
                } catch (Exception e) {}
            }).start();

            // Sending messages
            String msg;
            while ((msg = userInput.readLine()) != null) {
                out.println(msg);
            }

            socket.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}
```

# ▶️ **How to Execute**

### Step 1: Save files

- `ChatClient.java`
    

---

### Step 2: Compile

```bash
javac ChatServer.java
javac ChatClient.java
```

---

### Step 3: Run Server

```bash
java ChatServer
```

---

### Step 4: Run Client (new terminal or another system)

```bash
java ChatClient
```

---

Replace `"localhost"` in client with:

```java

```

# 🧠 **Explanation (Simple)**

### Server:

- Waits using `ServerSocket`
    
- Accepts connection
    
- Starts a thread → receives messages
    
- Main thread → sends messages
    

---

### Client:

- Connects to server
    
- Starts thread → receives messages
    
- Main thread → sends messages
    

---

# 🎓 **Important Points**

- TCP ensures **reliable delivery**
    
- Uses **Socket & ServerSocket**
    
- Threads enable **simultaneous communication**
    
- Works on same or different machines

# 🎯 **Viva Questions**

### 1. Why is TCP used here?

Because chat requires **reliable and ordered communication**.

---

### 2. Why use multithreading?

To allow **simultaneous send & receive**.

---

### 3. What happens without threads?

Chat becomes **one-way (blocking)**.

---

### 4. What is full duplex communication?

Both sides can **send and receive at the same time**.

---

### 5. Difference between this and echo program?

- Echo → one message reply
    
- Chat → continuous two-way communication
