# ✅ **Concept**

## 🔹 Stop-and-Wait Protocol

**Stop-and-Wait ARQ**

- Send **one frame**
    
- Wait for **ACK**
    
- If ACK lost → **retransmit**
    

👉 Simple but slower

---

## 🔹 Sliding Window Protocol

**Sliding Window Protocol**

- Send **multiple frames at once**
    
- Controlled by **window size**
    

👉 Faster and efficient

---

# 💻 **C Program**

```c
#include <stdio.h>
#include <stdlib.h>

void stopAndWait(int frames) {
    printf("\n--- Stop and Wait Protocol ---\n");

    int i = 1;

    while (i <= frames) {
        printf("Sending Frame %d\n", i);

        // Simulate ACK (80% success)
        if (rand() % 10 < 8) {
            printf("ACK received for Frame %d\n", i);
            i++; // move to next frame
        } else {
            printf("ACK lost! Resending Frame %d\n", i);
        }
    }
}

void slidingWindow(int frames, int windowSize) {
    printf("\n--- Sliding Window Protocol ---\n");

    int i = 1;

    while (i <= frames) {
        printf("\nSending window:\n");

        for (int j = 0; j < windowSize && (i + j) <= frames; j++) {
            printf("Frame %d sent\n", i + j);
        }

        printf("ACKs received for window\n");

        i += windowSize;
    }
}

int main() {
    int frames, windowSize;

    printf("Enter number of frames: ");
    scanf("%d", &frames);

    printf("Enter window size: ");
    scanf("%d", &windowSize);

    stopAndWait(frames);
    slidingWindow(frames, windowSize);

    return 0;
}
```

# 🧠 **Explanation (Simple)**

### Stop-and-Wait:

- One frame at a time
    
- Wait for ACK
    
- Retransmit if lost
    

### Sliding Window:

- Multiple frames sent
    
- Improves speed
    
- Uses window size
    

---

# 🎯 **Important Exam Points**

- Stop-and-Wait → low efficiency
    
- Sliding Window → high throughput
    
- ACK ensures reliability
    
- Used in data link layer
